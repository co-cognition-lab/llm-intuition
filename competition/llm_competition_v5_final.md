## 执行摘要

v4 构建了以"模型竞争"为轴心的 8 情景框架，将 LLM 竞争重新定义为"对人类直觉盲区的认知水平的竞争"。但 v4 遗留了一个结构性问题：**模型层只是价值链的一层**。入口控制（微信/抖音/鸿蒙）、平台锁定（Coze/Dify/MCP）、算力主权（昇腾/NVIDIA）——这些力量配置着模型竞争的边界条件，而 v4 将它们当作背景设定。

v5 引入**四层价值链模型**作为统一分析语言，将 8 个情景重新理解为"价值链上的 8 种应力"——每个情景对四层施以不同强度的压力，层与层之间通过供给关系传导。同时引入**贝叶斯信念追踪**：v4 是初始先验，v5 是第一次正式后验。

### v5 相对于 v4 的核心升级

| 维度 | v4 | v5 |
|------|:--|:--|
| 分析轴心 | 模型层竞争 | 四层价值链的权力分配 |
| 中国市场 | ~2 段概括 | 四层 × C/B 双轴完整分析 |
| 情景体系 | 8 个并列标签 | 8 种价值链应力，含层间传导路径 |
| 信念呈现 | 静态概率数字 | 先验→似然→后验追踪卡 |
| 证据标准 | 隐含 | 显式 L1-L5 证据等级体系 |

### 核心判断

1. **感知型直觉趋同速度超预期，模型层差异化窗口正在关闭**。编码和数学维度，开源三强与海外闭源前沿的差距已从"代际"压缩至"百分点"。模型能力本身不再是持久护城河。

2. **中国 C 端 Agent 入口已形成豆包/元宝双头格局，且入口集中度远超此前预期**。情景⑤（Agent 寡头化）概率上调至 12%。手机厂商的"零层入口"是唯一可能打破双头格局的外部力量，但瓶颈在商业而非技术。

3. **API 商品化拐点已触发**。DeepSeek 的 0.02 元/M 定价重新定义了模型层的单位经济基准。情景⑦（价格战挤压利润）概率上调至 15%。

4. **Agent 治理基础设施在灾难前建立**。ACS/EC-Council/ServiceNow 在 48 小时内同步涌现，Databricks 和 Google MCP 从平台层接入。但这只能降低事故概率，不能阻止构成性退化本身。

5. **人优先协议的战略价值正在被市场验证**。协议层标准化（MCP/A2A/ACS）和 Agent 治理基建的双重推进，正在将"人优先"从伦理选项升级为合规先发优势。

## 证据等级体系

本报告所有关键分析判断标注证据强度等级（L1-L5），适配 Co-Cognition Lab 通用证据框架，翻译到 P4 竞争情报场景：

| 等级 | 名称 | 定义 | 判据 |
|:--:|------|------|------|
| **L1** | 框架推演 | 从理论框架推导的分析判断，无外部事件佐证 | 仅靠逻辑推理 |
| **L2** | 类比外推 | 从其他行业/市场经验迁移 | 有外部参照物，非 LLM/Agent 领域直接证据 |
| **L3** | 单点信号 | 单一可验证事件或数据点支撑的判断 | ≥1 个可靠来源 |
| **L4** | 模式确认 | ≥2 个独立事件/来源形成一致方向 | 多个信号收敛，单个信号可验证 |
| **L5** | 因果验证 | 有明确反事实或对照证据，证明因果链条 | 极难达到——需时间线+动机链+排除替代解释 |

**使用原则**：大部分判断停留在 L3-L4。L1 判断（尤其退化理论和社会型直觉假设）是框架的核心支柱，但也是最可能被未来证据修正的部分。L5 级因果验证在竞争情报领域几乎不存在。

数据点的出处类型以缩写标注：`[官]` 一手官方源 / `[评]` 独立评测 / `[媒]` 媒体报道 / `[推]` 推测估算 / `[转]` 附件转引。

## 第一部分：分析框架

### §1.1 从模型竞争到价值链控制

v4 的核心问题是"LLM 竞争真正比拼的是什么"。我们的回答是：**比拼的是对人类直觉盲区的认知水平**。这个判断仍然成立。但 v4 的分析轴心是模型层——谁的大模型更强、谁的定价更低、谁的路线更有持续性。

v5 引入一个更基础的分析单元：**价值链**。

模型不是独立竞争的。Kimi K2.6 的能力再强，如果豆包的抖音入口把用户截流了，能力就没有兑现为市场权力。华为昇腾的算力再自主，如果 CANN 软件栈与 CUDA 的代际差距让开发者不愿迁移，自主就是纸面上的。Coze 的插件再多，如果 MCP 标准化让任何 Agent 都能接入同等工具库，插件就不是壁垒。

四层价值链模型试图捕捉这些跨层力量：

```
第四层：终端产品/入口层 ─ 用户直接交互的 Agent 产品
    ↕ 供给关系：第三层为第四层提供 Agent 开发和分发基础设施
第三层：平台/工具层 ─ Agent 开发、编排、分发的基础设施
    ↕ 供给关系：第二层为第三层提供模型 API
第二层：模型/基础设施层 ─ 大模型底座 + API 服务
    ↕ 供给关系：第一层为第二层提供训练和推理算力
第一层：硬件/算力层 ─ 训练与推理的物理基础
```

**层间关系**：下层支撑上层，上层依赖下层。层与层之间是供给关系，不是竞争关系。同层之间才是竞争关系。

**核心主张**：LLM 竞争不仅是"谁的模型最强"，而是"谁控制了价值链的哪个环节，以及控制力能否向上或向下传导"。

### §1.2 分析范围与边界

v5 的分析聚焦**中国市场**（四层均有丰富公开数据）。硅谷和欧洲仅作为对照基准出现——知道什么写什么，不知道就标"数据不足"。不要求对称覆盖。

每层内部按市场类型分为 C 端（消费者）和 B 端（企业）两条平行赛道。

### §1.3 情景作为价值链应力

v4 的 8 个情景不是并列的标签——它们是**价值链上的 8 种应力**。每个情景对各层施加不同强度的压力，通过供给关系跨层传导。

在 §3 的情景追踪卡中，每个情景标注"价值链应力分布"——一张热力图显示该情景在四层上的相对受力强度。这不是"把情景锁进某一层"，而是"理解情景在哪里产生、怎样传导、在哪层被放大或缓冲"。

### §1.4 贝叶斯信念追踪

v5 将 v4 的静态概率升级为动态信念追踪：

```
先验（v4 基期概率，2026年5月19日）
    ↓
似然（新观测：巡检信号 + 层分析发现）
    ↓
后验（v5 当前概率，2026年6月9日）
    ↓
下一轮关键监控项（什么信号会改变当前判断？）
```

这不仅是呈现方式的变化——这是认识论立场的变化。v4 说"我们认为概率是 X"。v5 说"我们此前认为概率是 X，最近观察到 Y，因此现在的判断是 Z。以下是可能让我们再次改变判断的事。"

**与 v4 的关系**：v4 不会被 v5 取代。v4 作为初始先验保留在网站上。v5 是第一次正式后验。两个版本并存，读者可以比较我们的判断如何随时间演进。

## 第二部分：价值链逐层分析

### §2.1 算力层

#### 格局概览

| 供应商 | 核心产品 | 主要客户 | 地缘风险 | 对中国 Agent 生态传导 |
|--------|------|------|:--:|------|
| NVIDIA | H100/B200/GB200 + CUDA | 中国大厂训练集群 | 高 — 出口管制持续 | 训练成本上升，推理依赖存量 |
| 华为 | 昇腾 910B/910C + CANN | 智谱、百度、政企 | 低 — 国产替代 | 软件栈差距制约迁移意愿 |
| 端侧 | 骁龙 NPU / 天玑 / 麒麟 | 手机厂商 | 低 | Agent 轻量任务可支撑 |

**C 端**：端侧芯片算力已能支撑轻量 Agent 任务（语音唤醒、短文本生成），7B 以下模型本地运行可行。瓶颈在模型压缩与算子适配，而非绝对算力。（L3·单点信号）

**B 端**：NVIDIA 仍主导训练，但获取渠道受限。GLM-5.1 基于 100,000 张昇腾 910B 完成训练，是目前唯一旗舰级"零 NVIDIA"案例 [官·智谱 AI，2026-04]。推理集群层面，存量 NVIDIA 芯片仍承载大部分 API 服务，昇腾推理实际占比 ⚠️ 缺乏公开数据。字节 2026 CAPEX 1,600 亿元中约半数投向半导体，但昇腾 vs NVIDIA 采购比例 ⚠️ 未拆分。（L4·模式确认——GLM+字节+腾讯三条独立算力投入线索）

#### 关键洞察

**双轨脱钩在训练层已不可逆，在推理层仍存缓冲**。GLM-5.1 证明旗舰模型训练可脱离 NVIDIA，但推理层存量 CUDA 生态的迁移成本使脱钩速度慢于训练层。如果美国进一步收紧管制，训练层冲击有限，B 端推理和微调将出现短期瓶颈。（L4·模式确认）

**端侧不是瓶颈，B 端推理是脆弱环节**。端侧供应链成熟，但高端 GPU 进口若完全中断，即时冲击在存量 NVIDIA 推理集群的维护和扩容。（L3·单点信号）

**华为昇腾软件栈对 Agent 部署"基本可用，体验有距"**。CANN/MindSpore 已支撑 GLM-5.1 训练，但第三方开发者社区规模、工具链成熟度与 CUDA 仍有代际差距。Agent 开发所需的动态图调试、分布式推理优化等高级功能 ⚠️ 缺乏独立评测验证。（L3·单点信号）

**对照硅谷/欧洲**：Google TPU v5p、AWS Trainium2、AMD MI300X 构成 NVIDIA 替代选项，但全球训练市场 NVIDIA 仍约 80% 份额 ⚠️ [推·行业估算]。欧洲无自主 AI 训练芯片。

### §2.2 模型层（中国）

> ⚠️ 本节聚焦中国模型层竞争。海外模型（GPT-5.5/Claude Opus 4.6/Gemini 3.0/Llama 4）在 §2.2 末尾"对照硅谷/欧洲"中简要对比。

#### 格局概览：开源三强 vs 闭源大厂

| 维度 | Kimi K2.6 | GLM-5.1 | DeepSeek V4-Pro | Qwen3.7-Max | 豆包 | 混元3.0 | 文心5.0 |
|------|------|------|------|------|------|------|------|
| 总/激活参数 | 1T/32B | 754B/~40B | 1.6T/49B | MoE/未公开 | ⚠️未公开 | ⚠️未公开 | ⚠️未公开 |
| 上下文 | 256K | 203K | **1M** | 1M | ⚠️未公开 | ⚠️未公开 | ⚠️未公开 |
| SWE-Bench Pro | **58.6%** | **58.4%** | ~55.4% | 强 | ⚠️未公开 | ⚠️未公开 | ⚠️未公开 |
| API 输入价/1M | $0.95 | $1.40 | **¥0.02** | 未公开 | 未公开 | 未公开 | 未公开 |
| 开源 | 权重开源 | **MIT** | 自定义 | API-only | 闭源 | 闭源 | 闭源 |

⚠️ 豆包、混元、文心的独立评测排名、SWE-Bench Pro 分数、激活参数均缺失。

#### 能力维度：感知型趋同加速

**编码**：Kimi K2.6（58.6%）与 GLM-5.1（58.4%）差距仅 0.2pp，双双超越 GPT-5.4（57.7%）和 Claude Opus 4.6（57.3%）。（L4·模式确认——月之暗面+智谱官方发布，2026-04）

**数学**：Kimi K2.6 AIME 2026 达 96.4%，GLM-5.1 达 95.3%，Qwen3.7-Max 达 95.1%——差距在 1-1.3pp 区间。（L3·单点信号）

**Agent 综合**：Kimi K2.6 73.1 分领先 GLM-5.1 65.3 分，差距仍显著（7.8pp）。（L3·单点信号）

**社会型直觉**：所有模型仍在"结构性不可达"——这不是性能差距，是具身性缺口。该边界在可预见未来（2026-2035）不会随规模扩大而消失。（L1·框架推演）

核心判断：**感知型直觉的趋同速度超预期，编码和数学维度已从"代际差距"压缩至"百分点级"**。模型能力本身不再是持久护城河。（L4·模式确认）

#### API 商品化：价格锚定效应已触发拐点

DeepSeek V4-Pro 2026-05-25 宣布 API 价格永久降至原价 25%，flash 版 0.02 元/M token。这不仅是降价——是重新定义了模型层的单位经济基准。当 API 定价低于企业自部署的电费+硬件折旧+运维成本时，API 从"差异化智能服务"滑向"大宗商品"。（L4·模式确认）

商品化不意味模型层消亡，而是利润池从"能力溢价"向"成本效率"和"附加服务"迁移。拥有自研芯片或算子级优化的厂商将在成本端建立新壁垒。（L2·类比外推）

#### DeepSeek 的基础设施悖论

DeepSeek 不做终端 Agent 产品，通过 API 被所有平台和产品调用——这使其成为 Agent 生态的"中立底座"。短期是差异化优势。长期看，当模型能力趋同后，没有入口和数据飞轮的纯 API 提供商将被上下游挤压——要么被平台封装为后台管道，要么被有自研模型的大厂替换。（L2·类比外推）

路线 C（全量开源+昇腾硬件绑定）是破解此悖论的最优解——将模型的软件标准影响力转化为硬件生态变现，类比 ARM 模式。

#### 对照硅谷/欧洲

GPT-5.5 和 Claude Opus 4.6 维持高溢价定价（$2.5-$10+/1M），但 DeepSeek 的价格锚定正在迫使海外厂商重新评估"模型能力溢价"的可持续性。当开源三强的感知型能力已逼近前沿，高溢价需靠"Agent 可靠性"和"社会型边界设计"支撑，而非模型分数本身。（L3·单点信号）

### §2.3 平台/工具层

#### 格局概览：开源社区 vs 大厂闭环

| 产品 | 定位 | 开源 | 生态锁定 | 价值链控制力 |
|------|------|:--:|------|:--:|
| **Coze 3.0** | C 端开发+分发 | 闭源 | 强 — 字节闭环 | 中（入口强，模型非顶尖） |
| **Dify** | B 端 LLM 平台 | 开源 | 中 — 20K+ Stars | 弱（社区壁垒） |
| **MaxKB** | B 端 RAG+Agent | 开源 | 中 — 100 万+ 安装 | 弱 |
| **华为云 Agent 引擎** | B 端企业平台 | 闭源 | 强 — 政企依赖 | 强（模型+入口+算力） |
| **其他 C 端** | LobsterAI/AutoClaw/QClaw | 混合 | 弱-中 | 弱 |

#### C 端：入口一体化 vs 独立工具

Coze 3.0 凭借 100+ 插件、多人多 Agent 协作和一键发布至抖音/微信/飞书，验证了"平台层入口一体化"的假设——开发即分发。（L3·单点信号）纯工具型平台面临被超级 App 截流的风险——当微信/抖音内嵌 Agent 开发能力时，独立平台的获客逻辑将被截断。（L2·类比外推）

#### B 端：开源生态锁定的低估价值

Dify（20K+ Stars）和 MaxKB（100 万+安装量）以社区规模形成软性锁定——开发者工作流沉淀和第三方集成构成迁移成本。但在企业市场，开源的卖点不是"免费"，而是"可 fork、可自托管"——数据主权和供应商锁定风险才是决策变量。（L4·模式确认）

#### 协议层对平台层的三重冲击

**冲击 1：工具集成价值被稀释**。MCP 标准化后，平台的"100+ 插件"不再是壁垒——任何 Agent 可通过 MCP 接入同等工具库。MCP 10K+ 服务器 vs Coze 100+ 插件，数量级差距显著。平台竞争从"谁集成更多"转向"谁的编排逻辑更优"。（L4·模式确认）

**冲击 2：编排价值被重新定义**。A2A 标准化 Agent 通信后，多 Agent 协作从专有技术变为协议兼容能力。但编排的深层价值——状态管理、错误恢复、人机边界设计——仍依赖平台实现。（L2·类比外推）

**冲击 3：安全管控从差异化变为标配**。ACS 定义了 6 个运行时钩子+Guardian Agent 机制。未来不具备 ACS 兼容性的 B 端平台可能失去政企投标资格。（L3·单点信号）

**综合**：协议层不是平台的终结者，而是准入门槛和成本结构的重塑者。平台必须从"协议适配者"升级为"编排+治理+体验"提供者。（L2·类比外推）

#### 对照硅谷/欧洲

LangChain 和 AutoGen 积极向 MCP/A2A 靠拢——从"框架"转型为"协议适配层"。Google 同时主导 A2A 和推 Vertex AI，是"协议+平台"双轨策略。（L3·单点信号）

### §2.4 终端产品/入口层

#### C 端：四国杀的入口解剖

|------|------|------|------|:--:|
| 豆包 | 自研 | 抖音 8亿+ | 2.3亿 [媒·QuestMobile] | 极强 |

| 通义千问 | Qwen3.7-Max | 钉钉+淘宝 | 1亿+ [官·阿里云] | 极强 |
| 文心一言 | 文心5.0 | 百度搜索 6亿+ | 2亿 [媒·WSJ] | 强 |

四家 MAU 均破亿。但入口类型决定 Agent 分发效率——不只是"有多少人看到"，而是"看到后有多少人会用、会持续用"。

**入口效率排序**：微信（社交信任传递）> 抖音（内容沉浸转化）> 淘宝（交易场景触发）> 百度搜索（检索→委托行为跨越）。

微信的社交关系链天然消解 Agent 采纳的最大障碍——信任。"你朋友在用的 Agent"是最强推荐引擎，元宝春节 DAU 5000 万+ 的爆发主要源于微信红包/群聊场景的社交裂变。（L3·单点信号）

**百度搜索作为入口的诚实评估**：百度搜索 6 亿+ MAU 是核心信息入口。但搜索是"人找信息"的主动行为，Agent 是"信息找人/自动执行"的混合行为。搜索→Agent 的转化率未公开——文心 2 亿 MAU 中多少来自搜索导流、多少来自独立 App，是未公开的关键变量。（L2·类比外推）

#### B 端：三条道路

| 产品 | 核心逻辑 | 差异化 |
|------|------|------|
| 钉钉 AI 助理 | 组织架构绑定 | 迁移成本极高——替换钉钉 = 重构审批流+知识库+组织架构 |
| WorkBuddy | 社交 CRM + 多模型接入 | 不绑定单一模型，企业可选 DeepSeek/GLM/Kimi |
| 华为云 Agent 引擎 | 政企安全合规 | 等保+鸿蒙+昇腾三端控制，硬性准入门槛 |

B 端竞争逻辑不同于 C 端：不是"流量→用户"的转化，而是"Agent 嵌入工作流而不被替换"的深度。三家分割不同客户类型——互联网公司偏好 WorkBuddy 的模型灵活性，传统企业偏好钉钉的组织绑定，政企必选华为。（L2·类比外推）

#### 手机厂商：零层入口

小米/OPPO/vivo/华为/荣耀拥有"零层入口"——系统级语音助手，触发摩擦低于任何 App。但模型短板是硬约束：自研模型未进入 BenchLM 前十。（L3·单点信号）

若手机厂商与 Kimi/DeepSeek 合作，可能出现"App 入口层（四国杀）+ 系统入口层（手机厂商+模型厂商联盟）"的第三种格局。障碍在商业（数据归属、品牌分成、用户关系谁持有）而非技术。（L1·框架推演）

华为是例外——盘古+鸿蒙+昇腾全栈自研，无需外部合作，但 C 端品牌认知度待提升。

#### 退化视角：入口越强，退化越深？

超级 App 入口降低 Agent 使用门槛，但也意味着用户"被动使用"而非"主动选择"。退化风险按入口类型递降：**微信（社交判断中介化）> 淘宝（消费决策外包）> 抖音（内容判断替代）> 百度搜索（信息检索代理）**。（L1·框架推演）

微信生态中的 Agent 对用户社会判断力的替代，比独立 Agent 产品更隐蔽、更不可逆——用户将"谁可信"的判断外包给微信生态 Agent 时，构成性退化的规模史无前例。12 亿用户的社会判断若被 Agent 中介，退化规模超过任何单一产品。（L1·框架推演）

如果微信 Agent 化未内置"人优先协议"，可能在退化暴露时面临监管急刹车——反而缩短 5-7 年的 Agent 化窗口。这是入口强度与退化风险正相关的核心悖论。

#### 对照硅谷/欧洲

海外无超级 App 入口。ChatGPT/Claude App 依赖独立下载与品牌认知；Gemini 依赖 Google 搜索+Android OS；Copilot 依赖 GitHub/Office 专业工具。中国 Agent 的获客成本由超级 App 承担，海外 Agent 需自建获客渠道。（L2·类比外推）

### §2.5 层间传导：权力在价值链中的流动

四层之间不是静态堆叠——权力在流动。以下方向值得关注：

**向上传导**（下层变化推动上层重构）：
- 算力脱钩（L1）→ 模型层双轨化（L2）→ 平台层需适配昇腾生态（L3）→ 终端层政企 Agent 被迫绑定华为（L4）
- MCP/A2A 标准化（L3）→ 平台层工具集成价值稀释（L3）→ 终端层 Agent 产品同质化加速
- 模型感知型趋同（L2）→ 模型层差异化消失 → 入口层成为唯一差异化来源 → 寡头化加速

**向下传导**（上层需求拉动下层供给）：
- 豆包/元宝 MAU 破亿（L4）→ 字节/腾讯模型推理需求暴增 → 算力投资加码
- 手机厂商零层入口激活（L4）→ 端侧 NPU 的 Agent 级算子适配需求 → 芯片设计方向调整

**阻断点**：
- 社会型直觉的结构性不可达（L2→L4 的硬边界）——模型能力向上传导存在天花板，微信的社交关系链不会因模型进步而被架空
- 政企安全合规（L4 的硬门槛）——B 端终端层由合规驱动，不受 C 端入口竞争逻辑影响

**管制性阻断**（新增类型，2026-06-26）：
- 2026 年 6 月，美国政府首次对已部署的商业 AI API（Fable 5）实施出口管制，附加国籍-based 访问控制与强制数据保留。这是一个此前未出现在 v5 框架中的传导类型——不是下层变化推动上层，也不是上层需求拉动下层，而是**外部政策力量同时对多层施加即时、强制性切断**
- 特征：政策层（非价值链内）→ API 层强制下线 + 国籍验证（第二层）→ 开源替代需求暴增（第二层替代供给）→ 政府-企业结盟竞争公开化（第三/四层）
- 如果国籍-based 访问控制从 Fable 5 扩展到其他前沿模型，这将成为价值链中最快的传导通道——不需要工程实现，一纸令下即可切断

**变化方向**：情景⑤（+8pp）和③B（+6pp）是上调最显著的两个。前者反映全球 AI 入口集中度远超此前预期——ChatGPT 9 亿周活 + 豆包/元宝双头格局 + DeepSeek $70 亿融资集中化。后者反映退化证据已从"叙事信号"升级为"量化趋势 + CEO 级确认"——Q1 科技裁员 78,557 人/47.9% AI 归因，三位 CEO 公开确认 AI 驱动的结构性裁员。情景①（-7pp）被多方向结构性压力挤压。权力从模型层向终端入口层的转移（⑤↑ ①↓）与退化风险的量化升级（③B↑）是同一枚硬币的两面。

### §4.2 三线程竞速

v4 提出了"中空期"的核心隐喻——Agent 能力在上升，人类能力在下降，中间是一个危险的窗口。v5 的层分析为这个隐喻增添了第三个线程：

**线程 1：Agent 能力上升**（模型层+平台层+终端层协同加速）
- 感知型能力趋近前沿（L4）
- MCP/A2A 标准化降低互操作成本（L4）
- 超级 App 入口使 Agent 触达 10 亿级用户（L4）

**线程 2：人类能力下降**（终端层退化风险）
- 入口越强退化越深——微信 12 亿用户的社会判断外包（L1）
- 退化是静默的——被 Agent 的常规良好表现掩盖（L1）
- Suleyman 18 个月的时间表表明速度竞赛在退化方（L3）

**线程 3：治理能力上升——但方向在偏移**（平台层+政策层双重驱动）
- ACS+Databricks+Google MCP 在灾难前建立治理基础设施（L4）——这是 5 月的判断
- **2026 年 6 月更新**：Fable 5 国籍锁事件揭示，实际发生的治理更多是**主权驱动的控制**（国籍验证、强制数据保留、生物识别），而非用户保护驱动的共生（L4·模式确认）
- 治理降低事故概率和刹车烈度（L3）——但主权治理的"保护"对象是国家边界，不是用户认知能力。这与 v4 假设的"治理防止退化暴露"存在偏差
- 治理仍然不能阻止人类认知外包——它拦截 Agent 行为，不拦截人类退化（L1）

**竞速状态（2026-06-26 更新）**：线程 1 领先，线程 2 在静默追赶。线程 3 在加速但不是 v4 预期的方向——它正在从"为防止退化而建"转向"为国家控制而建"。如果治理=主权控制，它对中空期的填充效果不确定：主权治理能阻止跨国 Agent 事故，但不能阻止退化本身的静默累积。三线程的相对速率仍然不可判定——但治理线程的方向偏移是 v5 发布以来最实质性的框架修正。

### §4.3 关键不确定性

以下判断的 L1-L2 等级意味着它们在未来 4-6 周内可能被新信号推翻：

1. **社会型直觉结构性不可达**（L1）：llm-intuition 理论的核心假设。如果出现模型在社会判断任务上达到人类水平的证据，该假设需要修正。

2. **感知型趋同速度**（L3→L4）：当前证据是多信号收敛的，但如果下一个模型版本（Kimi K3、GLM-6、GPT-6）出现非连续跳跃，趋同叙事将被推翻。

3. **入口强度与退化风险正相关**（L1）：纯框架推演。需要退化经验研究的验证（P4.10a，已移交 P1）。

4. **手机厂商合作的商业障碍 > 技术障碍**（L1）：框架推演。如果手机厂商宣布与模型厂商合作，该判断立即被推翻。

### §4.4 与 v4 的关系

v4（2026年5月19日）以模型层竞争为分析轴心，给出了 8 个情景的初始概率——这是我们的先验。

v5（2026年6月9日）将框架从一层扩展为四层价值链，引入中国 Agent 生态深度分析，并建立了结构化的贝叶斯信念更新机制——这是我们的第一次正式后验。

v4 不会被 v5 取代。它作为初始先验保留在网站上。两个版本并存，读者可以追踪我们的判断如何随时间演进——这正是贝叶斯更新框架的意义所在：不是"修正错误"，而是"基于新证据更新信念"。

## 第五部分：方法论与局限性

### §5.1 证据等级体系

证据等级体系的完整定义见本文开头部分。此处不再重复。

### §5.2 已知盲区与数据缺口

**跨层盲区**：

1. **闭源大厂模型能力的透明性缺失**：豆包/混元/文心的独立评测排名、架构细节、API 定价均未公开——模型层比较的三分之一对象处于不可见状态。
2. **平台层关键商业指标缺失**：Dify/MaxKB/Coze 的企业付费客户数、开发者留存率、Agent 部署量完全未公开——平台层竞争的核心指标不可观测。
3. **终端层用户行为的细粒度数据缺失**：四家 C 端产品的 DAU/MAU 比（仅豆包有数据）、用户画像、使用场景均未公开——无法判断"使用深度"。
4. **B 端市场的采购数据完全不透明**：企业客户数、续费率、使用深度均无法获取。
5. **C 端入口转化率不可知**：百度搜索→文心、抖音→豆包、微信→元宝的导流转化率未公开——入口效率排序的核心假设无法验证。

### §5.3 四层框架本身的局限性

四层价值链模型是一个分析框架选择，本身可争议（L1·框架推演）：

- **层边界是模糊的**：部分玩家跨越多层（华为：算力+模型+平台+终端；字节：模型+平台+终端），层模型在"跨层玩家"面前的分析力下降
- **双轴（C/B）的适用性不均**：算力层的 C/B 区分意义有限；终端层的 C/B 是两个完全不同的竞争逻辑，放在同一框架下有强行统一的嫌疑
- **硅谷/欧洲的数据不足**：全球版四层的对称分析不可行，本报告的"中国主场"策略是务实的限制而非设计选择

这些局限性不影响框架在当前分析范围内的有效性，但约束了框架向其他市场延伸的能力。
