## 这是什么

这是一个 **LLM 认知安全工具包**。

它不是传统 AI 安全。三个基本区别：

- **不是模型安全，是人机协同安全**——问题不在模型会不会出错，在人和模型的盲区是否重叠
- **不是漏洞扫描，是认知行为实验**——不是找 bug，是测试"AI 读到的信息是否被 silently 篡改"和"AI 是否跳过验证就行动"
- **不是最佳实践清单，是可证伪的操作手册**——不给 checklist，给可检验的假说和可追踪的信念更新路径

## 四个项目

### 钩子实验（感知层：信息保真）

在网页中嵌入对人类不可见但对机器可解析的隐藏指令，测试 7 个主流 LLM 的 web reader 管线行为。发现 43% 触发率，暴露了 LLM 在信息摄入阶段的不可见盲区。

当前状态：理论框架 v1.5，步 2 已闭合。 → [阅读全文](/zh/experiments/llm-hook)

### ITEC——指令触发型执行级联（执行层：操作安全）

给"AI 没问就做了"这个常见现象一个名字、一套分类框架、一个分析工具包。ITEC 不是 sycophancy（讨好用户）——是 LLM 在收到明确指令后系统性地跳过验证的**过程省略型失败**。

当前状态：论文框架 v1.0，3×2 诊断矩阵。 → [阅读全文](/zh/itec)

### 竞争格局 v5（战略层：行业格局）

用四层价值链模型（算力→模型→平台→终端）替代模型排名，对 8 个未来情景进行贝叶斯概率追踪。回答"LLM 行业权力如何分布与演变"——以及认知安全风险在产业链中如何被放大。

当前状态：v5 已发布（2026-06），循环巡检中。 → [阅读全文](/zh/competition-v5)

### 协同认知方法论（方法层：协防偏差）

把"用 AI 辅助思考"从直觉变成可重复、可检验、可改进的操作流程。包含五步探索框架、MERA 六问、失败模式分类学、三源交叉验证。不教你想什么——教你**怎么确保自己没有被 AI 带着走**。

当前状态：v2.3，基于六个子项目的 MERA 积累迭代。 → [阅读全文](/zh/methodology)

## 关于本 Lab

本工具包是 Co-Cognition Lab 的方向一产出（认知安全基础设施原型）。

Lab 的核心命题是**互补性**——AI 与人类各有直觉盲区，真正的风险不是一方超越另一方，而是双方的盲区在无人察觉处重叠。
