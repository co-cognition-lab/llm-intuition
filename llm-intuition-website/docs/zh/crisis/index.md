# 危机认知退化与防御：SIDA-F 框架 v1.4

> 14 个案例双向锚定——5 个失败案例揭示"认知在哪个环节断裂"，9 个成功案例标测"什么机制截断了退化"

## 一句话

极端情境下，组织与个体的认知功能不是变强——就是变弱。SIDA-F 告诉你**退化在哪个环节发生、如何级联放大、以及什么机制能截断它。**

## 退化如何发生：SIDA-F 五层环

危机认知遵循一个五层循环：感知(Sense)→解释(Interpret)→决策(Decide)→行动(Act)→反馈(Feedback)。每一层的退化都会级联放大下一层的退化。

**感知层退化**：信号被噪声淹没（福岛——200+条信息流无优先级）、信息在层级间异步（卡特里娜——通信物理摧毁后信息真空）、仪器在超出设计范围时系统性地欺骗（福岛——辐射计饱和显示"正常"）。

**解释层退化**：过早锁定判断（挑战者号——"O-ring侵蚀是可接受的"）、举证责任被翻转（深水地平线——"不能证明不安全"替代了"需要证明安全"）、整个解释框架无法修正（卡特里娜——联邦制响应框架在灾难性事件中不切换）。

**决策层退化**：权威压制质疑（三个核/航天案例全部出现——福岛的TEPCO总部→首相官邸过滤链）、沉没成本锁死（挑战者号——"已经推迟了这么多次，不能因为你一个Thiokol工程师就再推迟"）、决策在流程中消失（卡特里娜——Mission Assignment审批需要"无数签字"）。

**行动层退化**：协议在没有覆盖的情境下被僵化执行（**五案例全部出现**——消防队按常规火灾响应进入核辐射区、NRP框架在灾难性事件中不被打破）。

**反馈层退化**：反馈来得太晚（福岛——反应堆真实状态数周后才确认）、反馈永不可得（**五案例全部出现**——"如果当时选了B会怎样"永远无法被验证）、成功被错误编码为"系统安全"的证据（挑战者号——7次O-ring侵蚀但未爆炸=7次"安全"证据→累积了爆炸）。

## 防御如何运作：成功案例揭示的四条原理

**原理1：认知卸载——个体→结构。** 8/9成功案例不是通过"训练更强的个体"来防御——而是将认知负担从个体工作记忆转移到外部结构。航母甲板的彩色马甲让你不需要语言就知道谁在做什么。创伤团队的鉴别诊断穷举（"至少3个鉴别诊断"）让你不需要刻意记住不要漏——规则替你记了。

**原理2：格式决定可执行性。** 异议能否穿透权威梯度——取决于异议以什么格式被表述。技术参数格式（"传感器读数X，偏差Z%，超阈值"）→被接收为调用协议。个人意见格式（"我感觉不对"）→被接收为挑战权威。前者不需要勇气。后者需要——而在高权力距离的组织中，勇气常常不够。

**原理3：适当预载优于强行维持。** 在某些条件下，最优策略不是保护认知功能——是让认知功能不在危机中运行。日本三陆地区的"津波tendenko"——"地震→跑。不回头。不等家人。"——这个决定不是在海啸警报响起时做的。它是一个6岁孩子在学校反复听到的一句话。2011年东日本大地震。Kamaishi市2900名中小学生——几乎全部幸存。

**原理4：簇决定防御哲学。** 秒-分级操作（创伤团队、驾驶舱）→防御哲学=预载和自动化。小时-天级组织（指挥部、多机构响应）→防御哲学=结构和制度。同一个原则在两种时间尺度下需要完全不同的实现形态。

## 这个框架的诚实边界

1. **只检验了人类组织的危机认知——不覆盖AI。** 不是因为AI不存在，而是因为JEPA定理5.2已在数学上证明：非高斯隐变量（重尾、截断、相变——危机场景的数学特征）下AI无法学到可辨识的世界模型。危机认知中不可外包的判断责任——归根结底在人。

2. **14案例中只有簇2（小时-天级组织）的失败案例。** 簇1（秒-分级操作）和簇3（文化-具身演化）的退化模式基于理论推演，未经失败案例独立验证。

3. **这是一个诊断框架——不是一个预测框架。** 它能告诉你"这个组织在哪个认知环节最脆弱"——但不能告诉你"下一次危机会是什么"。
